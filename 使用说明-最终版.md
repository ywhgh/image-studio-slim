# 图片生成助手启动说明

这个项目有两个部分：

- 后端：Go 服务，默认地址 `http://127.0.0.1:8090`
- 前端：网页界面，默认地址 `http://127.0.0.1:3000`

普通本地运行时，需要开两个终端窗口：一个启动后端，一个启动前端。

现在后端支持后台排队生图：最多同时跑 3 个生图任务，超过 3 个会自动排队等待。

## 一、先准备环境

需要安装：

- Node.js 18 或更高版本，推荐 Node.js 20
- Go 1.22 或更高版本

检查是否安装成功：

Windows PowerShell：

```powershell
node -v
go version
```

Linux：

```bash
node -v
go version
```

## 二、Windows 启动方式

下面假设项目目录是：

```text
D:\chatgpt-main-image-studio
```

如果你的目录不同，把命令里的路径换成自己的项目路径。

### 1. 安装前端依赖

打开 PowerShell：

```powershell
cd D:\chatgpt-main-image-studio\frontend
corepack enable
corepack prepare pnpm@9.15.9 --activate
pnpm install
```

这一步只需要首次运行，或者依赖变更后再运行。

### 2. 启动后端

重新打开一个 PowerShell：

```powershell
cd D:\chatgpt-main-image-studio\server
go run .
```

如果你的 Go 固定安装在 `D:\go\bin\go.exe`，可以这样启动：

```powershell
cd D:\chatgpt-main-image-studio\server
D:\go\bin\go.exe run .
```

看到下面这行，说明后端启动成功：

```text
image-studio-slim listening on 0.0.0.0:8090
```

这个窗口不要关。

### 3. 启动前端

再打开一个 PowerShell：

```powershell
cd D:\chatgpt-main-image-studio\frontend
pnpm run dev
```

然后浏览器打开：

```text
http://127.0.0.1:3000
```

## 三、Linux 启动方式

下面假设项目目录是：

```text
/opt/chatgpt-main-image-studio
```

如果你的目录不同，把命令里的路径换成自己的项目路径。

### 1. 安装前端依赖

```bash
cd /opt/chatgpt-main-image-studio/frontend
corepack enable
corepack prepare pnpm@9.15.9 --activate
pnpm install
```

这一步只需要首次运行，或者依赖变更后再运行。

### 2. 启动后端

开一个终端：

```bash
cd /opt/chatgpt-main-image-studio/server
go run .
```

看到下面这行，说明后端启动成功：

```text
image-studio-slim listening on 0.0.0.0:8090
```

这个终端不要关。

### 3. 启动前端

再开一个终端：

```bash
cd /opt/chatgpt-main-image-studio/frontend
pnpm run dev
```

然后浏览器打开：

```text
http://127.0.0.1:3000
```

如果是在服务器上运行，浏览器访问：

```text
http://服务器IP:3000
```

## 四、Docker 启动方式

如果你只想部署，不想分别启动前后端，可以用 Docker。

Windows PowerShell：

```powershell
cd D:\chatgpt-main-image-studio
docker compose up -d --build
```

Linux：

```bash
cd /opt/chatgpt-main-image-studio
docker compose up -d --build
```

启动后打开：

```text
http://127.0.0.1:8090
```

如果是在服务器上运行，浏览器访问：

```text
http://服务器IP:8090
```

查看日志：

```bash
docker compose logs -f image-studio
```

停止服务：

```bash
docker compose down
```

## 五、页面里的外部中转怎么填

进入页面后，选择外部中转模式，然后填写：

```text
API 地址：https://api.openai.com/v1
API Key：你的 API Key
模型：gpt-image-1
```

如果用第三方中转，可以类似这样：

```text
API 地址：http://216.167.70.129:8317/v1
API Key：test
模型：gpt-image-2
```

## 六、常见问题

### 1. 页面能打开，但是生成时报 500

一般是后端没有启动。

Windows 检查：

```powershell
Invoke-RestMethod http://127.0.0.1:8090/api/v1/health
```

Linux 检查：

```bash
curl http://127.0.0.1:8090/api/v1/health
```

正常应该返回：

```json
{"code":0,"msg":"ok"}
```

### 2. 前端端口不是 3000

如果 `3000` 被占用，终端会显示一个新的地址。浏览器打开终端里显示的 `Local:` 地址即可。

### 3. pnpm install 很慢或失败

可以换成国内镜像：

```bash
pnpm config set registry https://registry.npmmirror.com
pnpm install
```

Windows PowerShell 也可以执行上面两行命令。

### 4. 提示 private/loopback hosts are not allowed

说明你的 API 地址是本机或内网地址，比如：

```text
http://127.0.0.1:8317/v1
http://192.168.x.x:8317/v1
```

如果确认这是你自己的中转服务，可以允许访问内网。

Windows：

```powershell
$env:ALLOW_PRIVATE_UPSTREAM="true"
go run .
```

Linux：

```bash
ALLOW_PRIVATE_UPSTREAM=true go run .
```

Docker 用户修改 `docker-compose.yml`：

```yaml
environment:
  ALLOW_PRIVATE_UPSTREAM: "true"
```

然后重新启动：

```bash
docker compose up -d --build
```

### 5. 生成很慢或提示 stream disconnected

这通常是上游生图服务或中转服务不稳定，不是前端问题。可以尝试：

- 稍后重试
- 换一个中转地址
- 降低分辨率
- 减少一次生成的图片数量

### 6. 怎么调整同时生图数量

默认最多同时跑 3 个生图任务。

Windows：

```powershell
$env:IMAGE_JOB_CONCURRENCY="3"
go run .
```

Linux：

```bash
IMAGE_JOB_CONCURRENCY=3 go run .
```

Docker 用户修改 `docker-compose.yml`：

```yaml
environment:
  IMAGE_JOB_CONCURRENCY: 3
```

一般不建议调太大，避免上游接口超时或被限流。

## 七、重新打包时建议排除

如果要把项目重新压缩发给别人，建议不要打包这些内容：

```text
frontend/node_modules
frontend/dist
*.log
*.exe
*.tsbuildinfo
```

别人拿到源码后重新执行 `pnpm install` 即可。
